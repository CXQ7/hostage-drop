Hostage-Drop (基于中间件的点对点文件交换实验)
本项目是一个基于 ActiveMQ 与 Spring Boot / Vue3 的课程实验作业。项目实现了一个带有“限时强制交换”功能的网页即时通讯程序，旨在通过具体的业务场景，练习和验证消息型中间件的核心特性。

🎯 实验功能与机制
文件暂存与交换：A 发送的文件不会直接传给 B，而是先作为消息发送到 ActiveMQ 的担保队列（Escrow Queue）中进行暂存。

限时解锁：B 收到通知后，必须在规定的时间窗口内上传自己的文件进行交换。后端校验通过后，才会将双方的文件互相派发。

超时自动清理：通过设置中间件消息的 TTL (Time-To-Live) 属性，如果 B 未能在倒计时内完成交换，暂存队列中的原文件将被中间件自动丢弃。

离线消息拉取：利用 ActiveMQ 的点对点队列持久化特性，如果接收方当前不在线，盲盒通知会保留在中间件中，待其上线连入 WebSocket 后自动拉取补偿。

🛠️ 技术栈
前端：Vue3 + Vite + Element Plus + @stomp/stompjs

后端：Spring Boot 3 + STOMP WebSocket

中间件：Apache ActiveMQ 5.x

🚀 启动与测试指南
在本地启动 ActiveMQ 服务（默认监听 tcp://localhost:61616）。

运行 Spring Boot 后端网关（默认端口 8080）。

运行 Vue 前端（默认端口 5173）。

测试建议：打开两个浏览器窗口，分别模拟用户 A 和 B 进行连接，互相发送文件以观察暂存队列与倒计时交换机制的流转过程。
